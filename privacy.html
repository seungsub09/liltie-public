Liltie 개인정보 처리방침

Liltie는 어떤 사용자 데이터도 수집·전송·공유하지 않습니다.

저장되는 정보

앱이 저장하는 모든 정보는 사용자의 기기에만 보관됩니다. iCloud 동기화를 사용 중인 경우, Apple의 CloudKit 개인 데이터베이스(본인 iCloud 계정 내부)로만 동기화됩니다.

  • 사용자가 기록한 음악 정보 — 트랙명·아티스트·앨범·커버 URL·메시지·무드 단계·날짜 → SwiftData (CloudKit 개인 DB와 자동 동기화 가능)
  • 앱 설정 — 테마, 돌아보기 애니메이션 등 → UserDefaults

Apple Music (선택)

Apple Music 연동은 선택 사항입니다. 사용자가 직접 연결을 허용한 경우에만 Apple Music 카탈로그 검색과 최근 들은 곡 정보를 조회합니다.

연동에 따른 처리는 Apple이 수행하며, Liltie는 결과 데이터(곡 메타데이터)만 화면에 표시합니다. Apple의 개인정보 처리방침을 따릅니다. 권한은 iOS 설정 또는 앱 내 설정에서 언제든 해제할 수 있습니다.

음악 검색 (iTunes Search API)

Apple Music 미연동 시, 음악 검색은 Apple의 공개 iTunes Search API를 통해 수행합니다. 요청에는 사용자가 입력한 검색어와 요청 국가(KR)만 포함되며, 사용자 식별 정보는 어떠한 형태로도 전송되지 않습니다.

검색 결과(트랙 정보, 미리듣기 URL, 앨범 커버 URL)는 사용자가 기록할 때만 기기에 저장됩니다.

업데이트 확인

Liltie는 새로운 버전 안내를 위해 Apple의 iTunes Lookup 공개 API(itunes.apple.com/lookup)에 앱 식별자로 조회 요청을 보냅니다. 이 요청에는 앱 식별자만 포함되며, 사용자 정보는 어떤 형태로도 포함되지 않습니다. 결과(게시된 최신 버전 번호)는 기기 내에 짧게 캐시되며 외부로 공유되지 않습니다.

외부 음원 앱 연결

사용자가 기록된 곡의 외부 링크 버튼을 누르면 Apple Music / Spotify / YouTube Music의 공개 검색 URL을 Universal Link로 호출합니다. 호출되는 URL에는 곡명과 아티스트명만 포함되며, 사용자 식별 정보는 포함되지 않습니다.

Instagram 스토리 공유

사용자가 공유 버튼을 누르면 카드 이미지를 시스템 Pasteboard에 복사하고 Instagram Stories의 공식 sharing API(instagram-stories://share)를 호출합니다. 이미지와 메타데이터는 시스템 Pasteboard를 통해 Instagram 앱으로만 전달되며, Liltie의 외부 서버로 전송되지 않습니다.

사용 권한

  • Apple Music (선택): Apple Music 카탈로그 검색 및 사용자 라이브러리(최근 들은 곡) 접근

권한은 iOS 설정 또는 앱 내 설정에서 언제든 켜고 끌 수 있습니다.

Liltie Privacy Policy

Liltie does not collect, transmit, or share any user data.

Information Stored

All information stored by the app is kept only on your device. If iCloud sync is enabled, data is synced exclusively to Apple's CloudKit private database (within your own iCloud account).

  • Music records you create — track name, artist, album, cover URL, message, mood level, date → SwiftData (auto-synced to your CloudKit private DB)
  • App settings — theme, lookback animation, etc. → UserDefaults

Apple Music (Optional)

Apple Music integration is optional. The app accesses Apple Music catalog search and your recently played songs only when you explicitly authorize the connection.

The processing is performed by Apple; Liltie only displays the resulting metadata. Apple's Privacy Policy applies. You can revoke permission at any time in iOS Settings or within the app.

Music Search (iTunes Search API)

When Apple Music is not connected, music search uses Apple's public iTunes Search API. The request contains only your search term and country code (KR); no user-identifying information is transmitted.

Search results (track metadata, preview URL, artwork URL) are saved to your device only when you create a record.

Update Check

Liltie queries Apple's public iTunes Lookup API (itunes.apple.com/lookup) using the app's identifier to check for new versions. This request contains only the app's own identifier and never includes any user information. The result (the latest published version number) is briefly cached on the device and is not shared externally.

External Music App Links

When you tap the external link button on a record, Liltie opens the public search URL of Apple Music / Spotify / YouTube Music via Universal Link. Only the track name and artist name are included in the URL; no user-identifying information is included.

Instagram Stories Sharing

When you tap the share button, the card image is copied to the system Pasteboard and Instagram Stories' official sharing API (instagram-stories://share) is invoked. Images and metadata are passed only to the Instagram app via the system Pasteboard and never to Liltie's external servers.

Permissions

  • Apple Music (optional): catalog search and access to your library (recently played)

You can toggle this at any time in iOS Settings or in the app settings.

Data Deletion

Individual records can be deleted from the menu (⋯) on each record screen.

Uninstalling the app removes all locally stored information. iCloud-synced data may remain in your iCloud account; manage it separately under iOS Settings > Apple ID > iCloud > Liltie.
